What Was Built

Triple Triad V2 is a fully playable, FFVIII-authentic browser implementation of the Triple Triad card mini-game. It features a rich single-player world mode, online two-player mode, and a polished UI built on React 19 + Vite + TypeScript.

Game Engine

Basic Capture
Compare touching card ranks; higher rank wins the cell.
Same Rule
Two or more matching rank pairs on adjacent sides → simultaneous capture.
Plus Rule
Two or more equal rank sums on adjacent sides → simultaneous capture.
Combo Rule
Same/Plus captures cascade — newly captured cards apply basic rule to neighbours.
Same Wall
Board edges count as rank 10 for Same rule evaluation.
Elemental Rule
Random board elements grant ±1 to all ranks for matching/mismatching cards.
Random Rule
5 cards dealt randomly from each player's full deck before the match.
Sudden Death
On draw, cards redistribute by ownership and the board resets. Loops until a winner.

World Mode

10 Progressive Areas
Balamb → Esthar unlocked sequentially. Each area has unique NPCs, rules, shops.
Per-NPC Card Pools
Each NPC draws from a fixed 10-card pool matching their area's difficulty tier.
Economy (Gil)
Win duels → earn gil. Spend at area shops to expand your card collection.
Win Tracking
NPC win counts displayed; rematch available from area panel.
Region Rules
Area-defined rule sets wired through to the engine (e.g. Galbadia uses Same + Plus).
Shop Deduplication
Already-owned cards shown with an "Owned" badge; re-purchase blocked.

UI / UX

Real Card Art
110 card images from /public/cards/{id}.png with rank overlays and element badges.
Card Animations
Spring scale-in on placement; 3D Y-axis flip (300 ms) on capture; hand pulse on active turn.
Win/Draw Overlay
Full-board result overlay with "Play Again" / "Return to World" actions.
Score Bar & Rules Bar
Live card count per player; active rule badges shown at top of board.
World Progress Bar
Areas unlocked / 10 with animated fill; newest area gets a gold pulse animation.
Ownership Colour Tint
Blue border (player) / red border (opponent) on all placed cards.

AI Opponents

  • Easy: Random valid move selection.
  • Medium: 1-ply greedy — maximises immediate board score.
  • Hard: Minimax with alpha-beta pruning, 4–6 ply (dynamic depth: deeper in late game when branching factor shrinks).

Known Limitations

  • Trade rules (Diff / Direct / All) not implemented. Only the "One" trade rule is in scope for V2. The other three are documented as future work.
  • No iOS / Safari testing. Development and testing used Chromium via Docker. CSS 3D transforms (card flip) are standard and should work, but no device testing was performed.
  • No audio. No sound effects or music. This was intentional — out of scope for V2.
  • Online multiplayer polish deferred. The two-player WebSocket mode works but was not polished in V2 (per Phase 1 scope decision).
  • Hard AI uses client-side minimax only. The AI runs in the browser main thread; for very early moves (45 × 40 × 28 × 24 branching at depth 4), response is near-instant with alpha-beta but a Web Worker would improve perceived responsiveness further.
  • Map images are placeholder. The world map and NPC portrait images are scaffolded but most real assets are absent; placeholder backgrounds are shown.

Suggested Future Expansions

  • Sound design: Card placement SFX, capture sting, win/loss fanfares matching the FFVIII OST aesthetic.
  • Diff / Direct / All trade rules: Complete the remaining three trade rules to match the full FFVIII rule set.
  • iOS / Capacitor build: The Capacitor config is already scaffolded (frontend/capacitor.config.ts); a native iOS build would require asset signing and App Store submission.
  • Leaderboards / statistics: Persistent win/loss record per NPC, global win rate, longest win streak.
  • Hard AI — Web Worker: Move the minimax search off the main thread to eliminate any UI jank on the first move of a hard game.
  • Rule abolition system: In FFVIII, rules can be abolished by playing in regions that have conflicting rule sets — a meta-game layer not yet implemented.
  • Card trading with NPCs: Post-match card trading (all trade rules) after adding Diff/Direct/All support.
